## 6/20 16:00 RFM分析彻底修复 — browser-use 截图验证通过

### 问题
用户反馈：RFM分层还是没有数据和图表

### 根因
`watch([sRef, pRef], ...)` 在 Vue 3 中监听模板 ref 的时序不可靠。watch 回调可能早于 ECharts DOM 容器完全渲染，导致 `echarts.init()` 拿到零尺寸容器。

### 修复方案
重写 RfmAnalysis.vue 的数据加载和图表渲染流程：
1. `const list = ref(D)` — demo 数据在 ref 声明时同步赋值（不变）
2. **移除 watch**，改用 `onMounted` + `setTimeout(() => draw(), 100)`
   - 100ms 延迟确保 `el-card` 等 Element Plus 组件完成内部渲染
3. ECharts `init()` 包裹 `try/catch`，防止任何异常导致页面崩溃
4. 添加调试状态条 `debug-banner`：绿色=已加载，红色=加载中
