## 七、启动方式

### 一键启动
双击项目根目录 `start-crm.bat`，自动：
1. 清理端口占用
2. 启动后端（Node.js + SQLite，端口8081）
3. 构建并启动CRM前端（端口3002）
4. 构建并启动会员门户（端口3003）

### 手动启动
```bash
# 终端1：后端
cd crm-server && node src/server.js

# 终端2：CRM前端
cd original-crm/crm-frontend && npx vite build && npx vite preview --port 3002

# 终端3：会员门户
cd member-portal && npx vite build && npx vite preview --port 3003
```

## 九、关键修复记录

| 类别 | 修复项 | 原因 |
|------|--------|------|
| **数据响应** | axios拦截器 `res.data` 解包后，50+页面仍取 `res.data.xxx` 导致 undefined | 批量修正为直接取 `res.xxx` |
| **API路径** | 商机/产品等模块缺少 `/list` 后缀 | 后端统一规范路径 |
| **SQLite** | 双引号字符串被当列名 + `value`/`month` 保留字 | 改单引号 + `NON_KEYWORDS` |
| **DDL** | 42+张新表缺失 + 字段不匹配 | 完整重建 |
| **RFM** | 图表无颜色（全局调色板对scatter无效） | 每series设 `itemStyle.color` |
| **AI助手** | 前端 `data = res.data` 取两次导致 undefined | 改为 `data = res` |
| **数据同步** | 会员注册不关联CRM客户 | 注册时自动创建关联 |
| **前端404** | baseURL含 `/api`，页面又加 `/api/` 前缀 | 去掉冗余前缀 |
